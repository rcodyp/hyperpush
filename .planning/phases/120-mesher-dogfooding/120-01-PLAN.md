<!-- Key Mesh syntax patterns for slot pipe and string interpolation -->
<!-- These are the ONLY new syntax forms being introduced — use exactly as shown -->

Slot pipe:
```
# Before: value passed as non-first argument (verbose)
let lower = String.to_lower(msg)
let no_hex = String.replace(lower, "0x", "")
String.trim(no_hex)

# After: slot pipe chains the value through (|2> inserts at position 2)
msg
  |> String.to_lower()
  |2> String.replace("0x", "")
  |> String.trim()
```

String interpolation (both ${} and #{} work — prefer #{} per v12.0 convention):
```
# Before: concatenation chain
println("[Mesher] Spike checker: escalated " <> String.from(n) <> " archived issues")

# After: interpolation (cleaner, no String.from needed for Int in #{})
println("[Mesher] Spike checker: escalated #{n} archived issues")
```

Key constraint: `#{expr}` evaluates expr inline. For Int values, `#{n}` produces the string form automatically. For complex expressions, wrap in parens: `#{(a + b)}`.

CRITICAL: Mesher uses `<>` for string concatenation. The ONLY substitutions allowed are:
1. Concatenation chains → #{...} interpolation (when it reads more naturally)
2. Non-first-arg function calls → slot pipe |N> (when it flattens the expression)
3. Multi-line string literals with many embedded newlines → heredocs (if any exist — check before applying)

Bar for updating: Only apply if the change is clearly more readable. Neutral or awkward → leave as-is.
