**authenticate_user** — Current: `Repo.query_raw(pool, "SELECT ... FROM users WHERE email = $1 AND password_hash = crypt($2, password_hash)", [email, password])`

Rewrite to:
```
pub fn authenticate_user(pool :: PoolHandle, email :: String, password :: String) -> User!String do
  let q = Query.from(User.__table__())
    |> Query.where(:email, email)
    |> Query.where_raw("password_hash = crypt(?, password_hash)", [password])
    |> Query.select(["id::text", "email", "display_name", "created_at::text"])
  let rows = Repo.all(pool, q)?
  if List.length(rows) > 0 do
    let row = List.head(rows)
    Ok(User { id: Map.get(row, "id"), email: Map.get(row, "email"), display_name: Map.get(row, "display_name"), created_at: Map.get(row, "created_at") })
  else
    Err("not found")
  end
end
```

NOTE: `Query.where_raw("password_hash = crypt(?, password_hash)", [password])` uses `?` as the parameter placeholder. The runtime `mesh_query_where_raw` stores this with a `RAW:` prefix and the SQL builder renumbers `?` to `$N`.

IMPORTANT: The `Query.select` function takes a `List<String>` not `List<Atom>` — use string field names. Check current usage in queries.mpl (line ~99 for existing patterns). If select with string list causes issues, use `Query.select_raw` instead.

Actually, `Query.select(Ptr, List<String>)` takes regular string field names. The `::text` casts may not work through the ORM's select (it quotes identifiers). Use `Query.select_raw(["id::text", "email", "display_name", "created_at::text"])` instead for the type casts.

```
pub fn create_session(pool :: PoolHandle, user_id :: String) -> String!String do
  # Step 1: Generate cryptographically random token via PG
  let token_rows = Repo.query_raw(pool, "SELECT encode(gen_random_bytes(32), 'hex') AS token", [])?
  if List.length(token_rows) > 0 do
    let token = Map.get(List.head(token_rows), "token")
    # Step 2: Insert session with ORM
    let fields = %{"token" => token, "user_id" => user_id}
    let _ = Repo.insert(pool, Session.__table__(), fields)?
    Ok(token)
  else
    Err("create_session: token generation failed")
  end
end
```

NOTE: The remaining `Repo.query_raw` is a pure function call (`SELECT encode(gen_random_bytes(32), 'hex') AS token`) with no table-specific data query — it's a PG utility call. The actual data INSERT is done through ORM `Repo.insert`.
