Lower_json_expr codegen strategy:
1. Start: `result = mesh_json_object_new()`
2. For each field in json_expr.fields():
   a. key_mir = MirExpr::StringLit(field.key_text(), MirType::String)
   b. val_mir = lower_expr(&field.value()) — gives raw Mesh value
   c. resolved_ty = self.types.get(&field.value().syntax().text_range())
   d. json_val = emit_json_value_for_type(val_mir, resolved_ty)
   e. result = mesh_json_object_put(result, key_mir, json_val)
3. Return: mesh_json_encode(result) [returns MirType::String]

emit_json_value_for_type dispatch:
```rust
fn emit_json_value_for_type(&mut self, expr: MirExpr, ty: &Ty) -> MirExpr {
    match ty {
        Ty::Con(c) if c.name == "Int" => call("mesh_json_from_int", vec![MirType::Int], expr),
        Ty::Con(c) if c.name == "Float" => call("mesh_json_from_float", vec![MirType::Float], expr),
        Ty::Con(c) if c.name == "Bool" => call("mesh_json_from_bool", vec![MirType::Bool], expr),
        Ty::Con(c) if c.name == "String" => call("mesh_json_from_string", vec![MirType::String], expr),
        Ty::Con(c) if c.name == "Nil" || c.name == "Unit" => {
            // nil literal — emit mesh_json_null()
            call_0("mesh_json_null", expr /*ignored*/)
        }
        Ty::Con(c) if c.name == "Json" || c.name == "String" => {
            // Already a Json ptr or String — if String, wrap; if Json ptr, pass through
            // For safety: if the resolved Ty is String (json { } returns String), pass as-is
            expr  // already the encoded string from a nested json { }
        }
        // For List types: use mesh_json_from_list with per-element callback
        // NOTE: This is complex — for Phase 132, simplify by only supporting
        // List<String> and List<Int> via the emit_collection_to_json helper.
        // For Option: check if sum type named "Option" and emit None->null, Some->value.
        // For structs with ToJson: call ToJson__to_json__StructName
        _ => {
            // Fallback: call Json.encode which dispatches via ToJson trait
            // Use existing emit_to_json_for_type helper (already in lower.rs)
            self.emit_to_json_for_type(expr, &self.ty_to_mir_type(ty), "json_literal")
        }
    }
}
```

NOTE: The simplest correct approach that handles 80% of cases and matches the CONTEXT.md spec:
- Use `self.emit_to_json_for_type(val_mir, &mir_ty, "json_literal")` as the dispatch. This existing helper already handles Int, Float, Bool (via their Json wrappers), String, Option, List, and ToJson structs.
- The `types` map on the lowerer stores `FxHashMap<TextRange, Ty>` — use it to get the inferred type.
- Look up how `generate_to_json_struct` calls `emit_to_json_for_type` to understand the pattern.

In lower_expr dispatch, add (after RegexExpr arm):

   c. Final: wrap result in `mesh_json_encode(result)` to get `MirType::String`.

**tests/e2e/json_literal_nested.mpl** — Nested json literal:
```mesh
fn main() do
  let inner = json { code: 200 }
  let outer = json { result: inner, ok: true }
  println(outer)
end
```
Expected: `{"result":{"code":200},"ok":true}` — but inner is a String (json encode output), so when it's put into the outer json object via `emit_to_json_for_type`, the String value will be wrapped as a JSON string (double-encoded).

IMPORTANT NOTE: Since `json { }` returns `String` (from `mesh_json_encode`), nesting a `json { }` result as a value in another `json { }` means the inner is a String that gets wrapped as a quoted JSON string (not embedded raw). This is a limitation of the String-first approach.

To handle nesting properly, use `Json.parse` approach or accept that nested json { } must be handled differently. For Phase 132, the simplest correct approach: if the value expression is itself a `json { }` literal, the executor should note that nesting will double-encode. Document this in the fixture with a comment and test with a workaround.
