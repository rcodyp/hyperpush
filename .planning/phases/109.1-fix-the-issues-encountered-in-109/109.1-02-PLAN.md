  <name>Task 1: Fix service loop argument loading to handle Bool, Float, and Struct types</name>
  <files>crates/mesh-codegen/src/codegen/expr.rs</files>
  <action>
**The bug:** In the service loop argument loading code (expr.rs lines 3509-3516), handler arguments are loaded from the message buffer using only two types:

```rust
let load_ty: BasicTypeEnum = if param_idx < handler_param_types.len()
    && handler_param_types[param_idx].is_pointer_type() {
    ptr_ty.into()
} else {
    i64_ty.into()
};
```

The sender side (`coerce_to_i64`, lines 3054-3096) correctly coerces all types to i64:
- Bool (i1): zero-extends to i64
- Float (f64): bitcast via alloca to i64
- Ptr: ptrtoint to i64
- Struct: alloca+load as i64

But the receiver side always loads as raw i64 for non-pointer types. When the handler function expects i1 (Bool) or f64 (Float), the LLVM `call` instruction gets an i64 where i1/f64 is expected — a type mismatch.

**The fix:** After loading the argument value as i64, convert it to the expected handler parameter type. Modify the service loop argument loading block (lines 3509-3520) to:

1. Keep the existing load-as-i64 or load-as-ptr logic
2. After loading, check `handler_param_types[param_idx]` and convert:
   - **Bool (i1)**: The loaded value is i64. Use `build_int_truncate(loaded_i64, i1_type, "trunc_to_bool")` to get the i1 value. This is the inverse of the `build_int_z_extend` in `coerce_to_i64`.
   - **Float (f64)**: The loaded value is i64. Use alloca+store i64 then load as f64 (bitcast via memory). This is the inverse of the Float path in `coerce_to_i64`.
   - **Struct**: The loaded value is i64. Use alloca+store i64 then load as the struct type. Only for structs that fit in 8 bytes (otherwise they'd be pointer-typed and already handled).
   - **Ptr**: Already handled by the existing `is_pointer_type()` check.
   - **i64 (Int, Pid)**: Already correct, no conversion needed.

The code should look approximately like:

```rust
let arg_val = self.builder
    .build_load(load_ty, arg_ptr, &format!("arg_{}", arg_idx))
    .map_err(|e| e.to_string())?;

// Convert loaded i64 to the expected handler parameter type.
let arg_val = if param_idx < handler_param_types.len() {
    let expected_ty = handler_param_types[param_idx];
    if expected_ty.is_int_type() {
        let int_ty = expected_ty.into_int_type();
        if int_ty.get_bit_width() < 64 {
            // Bool (i1) or other narrow int: truncate i64 → iN
            self.builder
                .build_int_truncate(arg_val.into_int_value(), int_ty, "trunc_arg")
                .map_err(|e| e.to_string())?
                .into()
        } else {
            arg_val
        }
    } else if expected_ty.is_float_type() {
        // Float: bitcast i64 → f64 via alloca
        let alloca = self.builder
            .build_alloca(self.context.i64_type(), "arg_f64_tmp")
            .map_err(|e| e.to_string())?;
        self.builder.build_store(alloca, arg_val).map_err(|e| e.to_string())?;
        self.builder
            .build_load(self.context.f64_type(), alloca, "arg_as_f64")
            .map_err(|e| e.to_string())?
    } else if expected_ty.is_struct_type() {
        // Small struct: bitcast i64 → struct via alloca
        let alloca = self.builder
            .build_alloca(self.context.i64_type(), "arg_struct_tmp")
            .map_err(|e| e.to_string())?;
        self.builder.build_store(alloca, arg_val).map_err(|e| e.to_string())?;
        self.builder
            .build_load(expected_ty, alloca, "arg_as_struct")
            .map_err(|e| e.to_string())?
    } else {
        arg_val
    }
} else {
    arg_val
};

handler_args.push(arg_val.into());
```

IMPORTANT: The `arg_val` from `build_load` is `BasicValueEnum`. When it's loaded as i64, `arg_val.into_int_value()` gives the `IntValue` needed for `build_int_truncate`. Verify the inkwell API types are correct.
