Modify TWO functions in mesher/storage/queries.mpl to use Crypto.uuid4() instead of Repo.query_raw gen_random_bytes calls. DO NOT touch create_user, authenticate_user, or any other function.

**create_api_key** (line ~97): Replace the two-step pgcrypto pattern with a single Crypto.uuid4() call. The current pattern generates "mshr_" + 48 hex chars. Replace it with:
```mpl
pub fn create_api_key(pool :: PoolHandle, project_id :: String, label :: String) -> String!String do
  # Generate API key using Crypto stdlib -- no DB round-trip needed
  let key_value = "mshr_" <> Crypto.uuid4()
  let fields = %{"project_id" => project_id, "key_value" => key_value, "label" => label}
  let _ = Repo.insert(pool, ApiKey.__table__(), fields)?
  Ok(key_value)
end
```
Note: Crypto.uuid4() gives 36 chars (UUID format with hyphens). "mshr_" + 36 chars = 41-char key. This changes key length from 53 chars (mshr_ + 48 hex) to 41 chars. This is acceptable — keys are stored as-is and compared as-is; length is not validated beyond presence. Update the comment to reflect the new format.

**create_session** (line ~191): Replace the two-step pgcrypto pattern with Crypto.uuid4() calls. To maintain 64-char token format, strip hyphens from two concatenated UUIDs:
```mpl
pub fn create_session(pool :: PoolHandle, user_id :: String) -> String!String do
  # Generate cryptographically random token using Crypto stdlib -- no DB round-trip needed
  # Two UUID4s with hyphens stripped = 32 + 32 = 64 hex chars (same format as before)
  let uuid1 = Crypto.uuid4() |2> String.replace("-", "")
  let uuid2 = Crypto.uuid4() |2> String.replace("-", "")
  let token = uuid1 <> uuid2
  let fields = %{"token" => token, "user_id" => user_id}
  let _ = Repo.insert(pool, Session.__table__(), fields)?
  Ok(token)
end
```
The slot pipe `|2>` inserts the piped value as the second argument of String.replace — same pattern used in fingerprint.mpl.

CRITICAL: Do NOT modify create_user (bcrypt crypt/gen_salt must stay in pgcrypto — no Mesh stdlib equivalent). Do NOT modify authenticate_user. Do NOT modify revoke_api_key (its timestamp generation is from PG now(), not randomness).
