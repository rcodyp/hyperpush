<task type="auto">
  <name>Task 1: Rewrite issue status transition and CRUD queries</name>
  <files>mesher/storage/queries.mpl</files>
  <action>
Rewrite the following 7 functions from Repo.execute_raw to ORM APIs:

1. **resolve_issue**: Replace `Repo.execute_raw(pool, "UPDATE issues SET status = 'resolved' WHERE id = $1::uuid AND status != 'resolved'", [issue_id])` with:
   - Build Query: `Query.from(Issue.__table__()) |> Query.where_raw("id = ?::uuid", [issue_id]) |> Query.where_raw("status != 'resolved'", [])`
   - Execute: `Repo.update_where(pool, Issue.__table__(), %{"status" => "resolved"}, q)`
   - Return: Extract count from result or return Ok(1) -- match existing return type `Int!String`
   - NOTE: Repo.update_where returns Result<Map<String,String>, String> per Phase 110 fix. The function needs to return Int!String, so map the result: `let _ = Repo.update_where(...)? \n Ok(1)`

2. **archive_issue**: Same pattern as resolve_issue but with `%{"status" => "archived"}` and WHERE `id = ?::uuid` (no status filter).

3. **unresolve_issue**: Same pattern with `%{"status" => "unresolved"}` and WHERE `id = ?::uuid`.

4. **discard_issue**: Same pattern with `%{"status" => "discarded"}` and WHERE `id = ?::uuid`.

5. **assign_issue**: Two branches:
   - If user_id length > 0: `Repo.update_where(pool, Issue.__table__(), %{"assigned_to" => user_id}, q)` where q has `Query.where_raw("id = ?::uuid", [issue_id])`
   - If empty: Use `Repo.execute_raw` for the NULL case since ORM map values are strings (cannot represent NULL in Map<String,String>). Keep `Repo.execute_raw(pool, "UPDATE issues SET assigned_to = NULL WHERE id = $1::uuid", [issue_id])` for the unassign branch ONLY. This is acceptable -- the ORM cannot express NULL through Map<String,String> values.

6. **delete_issue**: Two-step delete using Repo.delete_where:
   - Step 1: `let q_events = Query.from(Event.__table__()) |> Query.where_raw("issue_id = ?::uuid", [issue_id])` then `Repo.delete_where(pool, Event.__table__(), q_events)?`
   - Step 2: `let q_issue = Query.from(Issue.__table__()) |> Query.where_raw("id = ?::uuid", [issue_id])` then `Repo.delete_where(pool, Issue.__table__(), q_issue)`

7. **is_issue_discarded**: Replace Repo.query_raw with ORM query:
   - Build: `Query.from(Issue.__table__()) |> Query.where_raw("project_id = ?::uuid", [project_id]) |> Query.where(:fingerprint, fingerprint) |> Query.where(:status, "discarded") |> Query.select_raw(["1 AS found"])`
   - Execute: `Repo.all(pool, q)?`
   - Return: `Ok(List.length(rows) > 0)`

For all functions, preserve the exact function signatures (name, params, return type) so callers need no changes.
  </action>
  <verify>Run `meshc build mesher` -- should compile without errors. Verify no Repo.execute_raw calls remain for resolve_issue, archive_issue, unresolve_issue, discard_issue, delete_issue. The assign_issue unassign branch may retain one Repo.execute_raw for NULL.</verify>
  <done>7 issue status/CRUD functions rewritten to use Repo.update_where and Repo.delete_where. Zero Repo.execute_raw for status transitions except assign_issue NULL branch.</done>
</task>
