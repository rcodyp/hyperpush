**Critical bug in current fragment handling (repo.rs ~line 304-320):**
The fragment injection currently only replaces `?` with `$N`. But the requirement syntax is `Query.fragment("crypt($1, gen_salt('bf'))", [password])` which uses `$1` notation. Currently, `$1` in fragments is NOT renumbered -- it's emitted verbatim, which means if there are prior WHERE params using $1-$3, the fragment's $1 collides. Fix: detect `$N` patterns in fragment SQL and renumber them by adding the current param_idx offset.

Same bug exists in `where_raw` handling (~line 212-231) -- it only replaces `?` but not `$N`. However, where_raw was intentionally designed with `?` placeholders. For fragments, we need to support BOTH `?` and `$N` styles.

**Current fragment behavior** (repo.rs ~line 304-320):
- Fragment parts are appended to the END of the SQL string (after ORDER BY, LIMIT, OFFSET)
- This is wrong for the FRAG-02 requirement (fragments in WHERE, SELECT, ORDER BY, GROUP BY)
- `where_raw` already handles WHERE position correctly
- `select_raw` already handles SELECT position correctly
- Missing: ORDER BY and GROUP BY fragment positions

**Solution design:**
1. Fix `$N` renumbering: regex or char-by-char scan for `$` followed by digits, replace with `$(N + offset)`
2. Add `mesh_query_order_by_raw(q, expression)` -- stores expression with "RAW:" prefix in order_fields
3. Add `mesh_query_group_by_raw(q, expression)` -- stores expression with "RAW:" prefix in group_fields
4. Update ORDER BY SQL generation to handle "RAW:" prefix (emit verbatim)
5. Update GROUP BY SQL generation to handle "RAW:" prefix (emit verbatim)
6. Fix fragment $1 renumbering in both `where_raw` and `fragment` code paths

**1. Add renumber_dollar_params helper in repo.rs** (before build_select_sql_from_parts):
Create a helper function that scans a string for `$N` patterns and renumbers them:
```rust
/// Renumber $N placeholders in a SQL fragment.
/// E.g., with offset=3: "$1" -> "$4", "$2" -> "$5"
/// Also replaces `?` with next sequential $N.
/// Returns (renumbered_sql, number_of_params_consumed).
fn renumber_placeholders(sql: &str, start_idx: usize) -> (String, usize) {
    let mut result = String::with_capacity(sql.len());
    let mut max_placeholder = 0usize;
    let mut question_count = 0usize;
    let chars: Vec<char> = sql.chars().collect();
    let mut i = 0;
    while i < chars.len() {
        if chars[i] == '?' {
            let new_idx = start_idx + question_count;
            result.push_str(&format!("${}", new_idx));
            question_count += 1;
            i += 1;
        } else if chars[i] == '$' && i + 1 < chars.len() && chars[i + 1].is_ascii_digit() {
            // Parse the number after $
            let mut num_str = String::new();
            let mut j = i + 1;
            while j < chars.len() && chars[j].is_ascii_digit() {
                num_str.push(chars[j]);
                j += 1;
            }
            if let Ok(n) = num_str.parse::<usize>() {
                if n > max_placeholder {
                    max_placeholder = n;
                }
                let new_idx = start_idx + n - 1; // $1 -> $start_idx, $2 -> $start_idx+1
                result.push_str(&format!("${}", new_idx));
            } else {
                result.push('$');
                result.push_str(&num_str);
            }
            i = j;
        } else {
            result.push(chars[i]);
            i += 1;
        }
    }
    let params_consumed = if question_count > 0 { question_count } else { max_placeholder };
    (result, params_consumed)
}
```

    }
}
