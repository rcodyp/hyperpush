## Task 4: Run benchmarks on Fly.io (STILL PENDING — needs human action)

Machines from previous run may still exist. Check first:
```bash
fly machine list -a bench-mesh
```

If old machines exist, destroy them:
```bash
fly machine destroy <id> -a bench-mesh -f
```

### Deploy fresh servers machine:
```bash
# From repo root:
fly machine run . -a bench-mesh \
  --name bench-servers \
  --dockerfile benchmarks/fly/Dockerfile.servers \
  --vm-size performance-2x \
  --region ord \
  --restart no \
  --detach
```

### Wait for "All servers running" in logs:
```bash
fly logs -a bench-mesh --machine <servers-id>
# Look for: "=== All servers running ==="
```

### Get server IPv6:
```bash
fly machine list -a bench-mesh
# Note the IPv6 address (fdaa:...) for bench-servers
```

### Run load generator:
```bash
fly machine run . -a bench-mesh \
  --name bench-loadgen \
  --dockerfile benchmarks/fly/Dockerfile.loadgen \
  --vm-size performance-2x \
  --region ord \
  --restart no \
  --detach \
  -e SERVER_HOST="[<servers-ipv6>]"
```

### Collect results:
```bash
# Stream loadgen results:
fly logs -a bench-mesh --machine <loadgen-id>

# Collect RSS data from servers:
fly logs -a bench-mesh --machine <servers-id> --no-tail | grep '^RSS,'

# Get runtime versions (SSH into servers):
fly ssh console -s -a bench-mesh -C "go version && rustc --version && elixir --version && meshc --version"
```

### Provide back to Claude:
1. Full results table (req/s, p50, p99 per language × endpoint)
2. Peak RSS per language (from RSS, lines)
3. Runtime versions
4. Region used (ord)
5. Machine spec (performance-2x = 2 vCPU, 4GB RAM)

<decisions_made>

- Mesh /json crash was socat exhaustion, NOT a string escape compiler bug
- String escape handling in compiler is correct (unescape_string() works properly)
- Eliminated socat entirely — direct [::] binding is cleaner and more reliable
- Used `hey` (Go HTTP load tester) instead of wrk — wrk has no IPv6 support
- Fly 6PN is IPv6-only; with bindv6only=0, [::] accepts both IPv4+IPv6
- SERVER_HOST must use bracket notation for IPv6 in URLs: [fdaa:3:...]
- Elixir 1.16 (not 1.17+) required for OTP 24 compatibility
- All 4 servers now bind directly on [::]:port — no proxy layer
</decisions_made>
