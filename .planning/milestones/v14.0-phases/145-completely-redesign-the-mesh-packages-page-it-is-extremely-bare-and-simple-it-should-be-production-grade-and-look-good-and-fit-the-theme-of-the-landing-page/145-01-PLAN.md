    Fix 2 of 3 confirmed API bugs (the third — `pkg.latest_version` vs `pkg.version` — is in the Svelte template, fixed in Plan 02):

    **Bug 1 — Search query param mismatch**: `packages-website/src/routes/search/+page.server.js` sends `?q=` but the registry `SearchParams` struct only reads `params.search`. Fix: send `?search=` to the registry API while keeping the form URL using `?q=` (so `/search?q=term` works as a URL). Rewrite the loader:
    ```js
    export async function load({ fetch, url }) {
      const q = url.searchParams.get('q') || '';
      if (!q.trim()) return { packages: [], query: q };
      try {
        const res = await fetch(`https://api.packages.meshlang.dev/api/v1/packages?search=${encodeURIComponent(q)}`);
        if (!res.ok) return { packages: [], query: q, error: 'Registry unavailable' };
        const packages = await res.json();
        return { packages, query: q };
      } catch {
        return { packages: [], query: q, error: 'Search failed' };
      }
    }
    ```

    **Bug 2 — Missing versions HTTP endpoint**: The DB function `db::packages::list_versions()` exists but no HTTP route exposes it. The old `packages/[name]/+page.svelte` references `data.pkg.versions` which doesn't exist in the API response.

    In `registry/src/routes/metadata.rs`, add a new handler after `version_handler`:
    ```rust
    #[derive(Serialize)]
    pub struct VersionListItem {
        pub version: String,
        pub published_at: chrono::DateTime<chrono::Utc>,
        pub download_count: i64,
        pub size_bytes: i64,
    }

    /// GET /api/v1/packages/{name}/versions
    /// Returns all versions for a package ordered newest first.
    pub async fn versions_handler(
        State(state): State<Arc<AppState>>,
        Path(name): Path<String>,
    ) -> Result<Json<Vec<VersionListItem>>, AppError> {
        // Ensure package exists (returns 404 if not)
        db::packages::get_package(&state.pool, &name)
            .await
            .map_err(|e| AppError::Internal(e.to_string()))?
            .ok_or(AppError::NotFound)?;
        let versions = db::packages::list_versions(&state.pool, &name)
            .await
            .map_err(|e| AppError::Internal(e.to_string()))?;
        Ok(Json(versions.into_iter().map(|v| VersionListItem {
            version: v.version,
            published_at: v.published_at,
            download_count: v.download_count,
            size_bytes: v.size_bytes,
        }).collect()))
    }
    ```

    In `registry/src/routes/mod.rs`, register the new route. Place it BEFORE the `/{name}/{version}` route to prevent Axum matching "versions" as a version string:
    ```rust
    .route("/api/v1/packages/{name}/versions", get(metadata::versions_handler))
    .route("/api/v1/packages/{name}/{version}", get(metadata::version_handler))
    ```

    Run `cargo check` in registry/ to confirm it compiles.
