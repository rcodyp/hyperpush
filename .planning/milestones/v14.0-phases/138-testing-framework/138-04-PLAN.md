<objective>
Fix the depth-tracking bug in `emit_non_test_items` in `test_runner.rs` so that describe
blocks containing setup/teardown sub-blocks are correctly suppressed from the preprocessed
output. Add an e2e fixture that exercises setup/teardown and verifies the fix.

Gap: When `emit_non_test_items` processes a describe block that contains `setup do...end`
or `teardown do...end` sub-blocks, the describe() call leaks into the preprocessed output.
The Mesh type checker then sees a `describe(...)` call without its body argument and rejects
it with "expected 2 arguments, found 1". Describe blocks WITHOUT setup/teardown work correctly.

Purpose: Unblock TEST-07 — shared setup/teardown for describe blocks.
Output: Fixed `test_runner.rs` + new `tests/e2e/test_setup_teardown.test.mpl` fixture.

// Existing token types and lexer (already in test_runner.rs):
enum TToken {
    TestKw, DescribeKw, SetupKw, TeardownKw,
    Do, End, Fn, If, While, Case, For, Actor, Service, Receive,
    StringLit(String), LParen, RParen, Other(String),
}

fn tokenize_test_source(source: &str) -> Vec<TToken>;

// The replacement emit_non_test_items should use tokenize_test_source()
// instead of the character-level count_do_in_line / count_end_in_line approach.
// The token-based approach correctly handles nested blocks without string-literal interference.

// Depth-opening tokens (tokens that introduce a block requiring a matching 'end'):
//   Do, If, While, Case, For, Receive
// Depth-closing tokens:
//   End
// Note: Fn does NOT open a depth by itself — the Do that follows it does.
// TestKw / DescribeKw / SetupKw / TeardownKw do NOT open depth — they
// appear before the Do that actually opens their block.

// The new emit_non_test_items algorithm:
// 1. tokenize_test_source(source) → tokens
// 2. Walk tokens maintaining depth (i64, starts at 0)
// 3. When depth==0 and we see TestKw or DescribeKw:
//    - Enter "skip mode": skip all tokens until the matching End
//    - Depth tracking: every Do (and If/While/Case/For/Receive) inside increments
//      skip_depth; every End decrements; when skip_depth hits 0, exit skip mode
//    - Do NOT emit the TokenKw or anything in skip mode
// 4. When depth==0 and we see other tokens: emit them verbatim
// 5. When depth > 0 (inside a non-test block): emit all tokens and track depth

// Key: Do NOT emit tokens while in skip mode (when processing test/describe subtrees).
// The skip_depth starts at 1 when entering skip mode (because the Do following
// test/describe is the opening of the block that needs to be skipped).

// token_to_str() already exists in the file for converting TToken back to source text.

  <name>Task 1: Rewrite emit_non_test_items using the token-based lexer</name>
  <files>compiler/meshc/src/test_runner.rs</files>
  <action>
    Replace the `emit_non_test_items` function and its two helper functions
    (`count_do_in_line`, `count_end_in_line`) with a token-based implementation
    that reuses `tokenize_test_source` (already in the file).

    **New implementation strategy:**
    1. Call `tokenize_test_source(source)` to get the token stream.
    2. Walk the token stream tracking two pieces of state:
       - `emit_depth: usize` — depth of non-test blocks we are currently EMITTING
         (0 = top level; > 0 = inside a user fn/struct/etc. that we are emitting)
       - `skip_depth: usize` — depth of test/describe block we are SKIPPING
         (0 = not skipping; > 0 = inside a test/describe subtree being suppressed)
    3. Rules:
       - If `skip_depth > 0`: we are inside a test/describe block; skip ALL tokens
         and track depth to find the matching end.
         - `Do | If | While | Case | For | Receive` → `skip_depth += 1`
         - `End` → `skip_depth -= 1`. If it hits 0, exit skip mode. Do NOT emit this End.
         - Everything else → skip (do not emit)
       - If `skip_depth == 0` and `emit_depth == 0` (top level, not skipping):
         - `TestKw | DescribeKw` → enter skip mode: `skip_depth = 1`.
           Then in the same token position, continue with the next token in skip mode.
           (The TestKw/DescribeKw token itself is NOT emitted.)
           **Crucially**: skip_depth starts at 0 and only increments when we hit the Do.
           Wait, actually: when we see TestKw, we want to skip the entire block from
           TestKw through its matching End. Set a flag `in_skip = true`. Skip the test
           label (StringLit), the Do (which increments skip_depth to 1), and all content
           until skip_depth returns to 0 when the matching End is consumed.
           **Simplest approach**: set `skip_depth = 1` when we see TestKw/DescribeKw
           AND we've confirmed the next Do is coming. Alternative: set skip_depth = 0
           in a "looking for opening do" state. The cleanest approach:
           Use a boolean `skip_until_end: bool` to mean "skip the current test/describe
           block starting from the Do we haven't seen yet". When TestKw/DescribeKw at
           depth 0, set `skipping = true`. Continue reading tokens WITHOUT emitting.
           When we hit Do while `skipping=true` and `skip_depth==0`: set `skip_depth=1`,
           continue (already set to not emit). When skip_depth > 0, track Do/End as above.
           When skip_depth reaches 0: `skipping = false`, done with this block.
         - All other tokens (Fn, Other, StringLit, etc.) → emit via `token_to_str`
       - If `skip_depth == 0` and `emit_depth > 0` (inside a user-defined block):
         - `Do | If | While | Case | For | Receive` → `emit_depth += 1`, emit token
         - `End` → `emit_depth -= 1`, emit token
         - All others → emit token
         - Special: if at `emit_depth > 0`, do NOT check for TestKw/DescribeKw
           (they can't be top-level inside a user fn)

    **Concrete implementation:**

    ```rust
    fn emit_non_test_items(source: &str, out: &mut String) {
        let tokens = tokenize_test_source(source);
        let mut i = 0;
        let mut emit_depth: usize = 0;   // depth inside user-defined blocks (fn, struct, etc.)
        let mut skipping: bool = false;   // true when we're suppressing a test/describe block
        let mut skip_depth: usize = 0;   // block depth inside the skipped test/describe block

        while i < tokens.len() {
            let tok = &tokens[i];

            if skip_depth > 0 {
                // Inside a test/describe block body — skip everything and track nesting.
                match tok {
                    TToken::Do | TToken::If | TToken::While
                    | TToken::Case | TToken::For | TToken::Receive => {
                        skip_depth += 1;
                    }
                    TToken::End => {
                        skip_depth -= 1;
                        if skip_depth == 0 {
                            skipping = false;
                        }
                    }
                    _ => {}
                }
                i += 1;
                continue;
            }

            if skipping {
                // Between TestKw/DescribeKw and the opening Do (skipping label, parens, etc.)
                // Once we see Do, start depth tracking.
                if matches!(tok, TToken::Do) {
                    skip_depth = 1;
                }
                // Do not emit anything while skipping.
                i += 1;
                continue;
            }

            // Not skipping. emit_depth tracks depth of user-defined blocks.
            match tok {
                TToken::TestKw | TToken::DescribeKw if emit_depth == 0 => {
                    // Start of a test/describe block at top level — suppress it.
                    skipping = true;
                    // Do not emit the keyword.
                }
                TToken::Do | TToken::If | TToken::While
                | TToken::Case | TToken::For | TToken::Receive => {
                    emit_depth += 1;
                    out.push_str(&token_to_str(tok));
                }
                TToken::End => {
                    if emit_depth > 0 {
                        emit_depth -= 1;
                    }
                    out.push_str(&token_to_str(tok));
                }
                _ => {
                    out.push_str(&token_to_str(tok));
                }
            }
            i += 1;
        }
    }
    ```

    After replacing `emit_non_test_items`, also DELETE the now-unused helper functions
    `count_do_in_line` and `count_end_in_line` entirely (they are only called from
    `emit_non_test_items`). This avoids dead_code warnings.

    **Important correctness notes:**
    - `TestKw | DescribeKw if emit_depth == 0`: the `if emit_depth == 0` guard ensures
      that if someone names a variable `test` inside a user fn (unlikely but valid),
      it is not misidentified as a top-level test block.
    - The algorithm correctly handles: describe blocks without setup/teardown, describe
      blocks WITH setup/teardown (because setup/teardown do NOT affect skip_depth — they
      have their own Do/End pairs which are tracked by skip_depth normally), describe
      blocks with nested test blocks (each test's Do/End contributes to skip_depth),
      and top-level test blocks.
    - User-defined functions (fn definitions) use emit_depth to track that we are inside
      them, so their content is emitted correctly.

    ```
      end

      end
    end

      assert(1 + 1 == 2)
      assert_eq("foo", "foo")
    end
    ```

    **Design notes:**
    - `setup do ... end` uses local `let` bindings as side effects. The setup body IS
      inlined into each test body function by the preprocessor (the `setup_body` is
      emitted before the test body in `fn __test_body_N()`). So `let base = 100` inside
      setup followed by `assert(100 + 5 == 105)` in the test body works because both
      are in the same generated function.
    - The teardown body is emitted AFTER the test body in `fn __test_body_N()`. Keep
      teardown bodies simple (no assertions that would affect pass/fail counts).
    - Use `let _ = 0` in teardown as a no-op to exercise the teardown path without
      side effects that could interfere with test pass/fail.
