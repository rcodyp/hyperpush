<objective>
Implement the assertion helper runtime functions and wire them through all 5 compiler
registration points. This creates the foundation that Plan 03's DSL lowering will call.

The runtime provides `mesh_test_assert`, `mesh_test_assert_eq`, `mesh_test_assert_ne`,
`mesh_test_assert_raises`, `mesh_test_begin`, `mesh_test_pass`, `mesh_test_fail_msg`,
`mesh_test_summary`, and `mesh_test_cleanup_actors` as `extern "C"` functions in
`mesh-rt/src/test.rs`.

Purpose: Without the runtime, the test DSL lowering in Plan 03 would have nothing to call.
The runtime maintains per-test state (pass/fail counts, current test name) and handles the
`catch_unwind` for `assert_raises`. Failure messages are accumulated and reprinted at the
bottom of the run (per user decision).

Output: `mesh-rt/src/test.rs`, updated `lib.rs`, updated `infer.rs`, `lower.rs`, `intrinsics.rs`
</objective>

    ```rust
    thread_local! {
        static PASS_COUNT: Cell<i64> = Cell::new(0);
        static FAIL_COUNT: Cell<i64> = Cell::new(0);
        static CURRENT_TEST: RefCell<String> = RefCell::new(String::new());
        static QUIET_MODE: Cell<bool> = Cell::new(false);
        // Accumulates all failure messages for end-of-run reprint
        static FAIL_MESSAGES: RefCell<Vec<String>> = RefCell::new(Vec::new());

    }
    ```

    **ANSI colors:**
    ```rust
    const GREEN: &str = "\x1b[32m";
    const RED:   &str = "\x1b[31m";
    const BOLD:  &str = "\x1b[1m";
    const DIM:   &str = "\x1b[2m";
    const RESET: &str = "\x1b[0m";
    ```

    **Helper to read MeshString as &str:**
    ```rust
    unsafe fn mesh_str(s: *const crate::string::MeshString) -> &'static str {
        let ms = &*s;
        std::str::from_utf8_unchecked(std::slice::from_raw_parts(ms.ptr, ms.len as usize))
    }
    ```

    **Functions to implement:**

    `mesh_test_begin(name: *const MeshString)` — called before each test body:
    - Store test name in CURRENT_TEST
    - In verbose mode (not quiet): print `"  {DIM}running:{RESET} {name}"` (on same line, no newline yet)
    - Flush stdout

    `mesh_test_pass()` — called after test body completes without panic:
    - Increment PASS_COUNT
    - In verbose mode: overwrite or follow the "running:" line with `"\r  {GREEN}✓{RESET} {name}\n"`
    - (Use `\r` to overwrite the running line, then reprint with green checkmark)

    `mesh_test_fail_msg(msg: *const MeshString)` — called on assertion failure:
    - Increment FAIL_COUNT
    - Get the current test name from CURRENT_TEST
    - In verbose mode: print `"\r  {RED}✗{RESET} {name}\n"` + `"    {RED}{msg}{RESET}\n"`
    - Accumulate the full failure entry into FAIL_MESSAGES for the end-of-run reprint:
      ```rust
      let entry = format!("  {RED}{BOLD}✗{RESET} {name}\n    {RED}{msg}{RESET}",
          name = current_test_name,
          msg = msg_str);
      FAIL_MESSAGES.with(|fm| fm.borrow_mut().push(entry));
      ```
      This ensures `mesh_test_summary` can reprint all failures at the bottom of the run.

    `mesh_test_assert(cond: i8, expr_src: *const MeshString, _file: *const u8, _file_len: u64, _line: u32)`:
    - If cond == 0: call mesh_test_fail_msg with message `"assert failed: {expr_src}"`
    - Then call mesh_panic to unwind (so the test harness can catch it)
    - Note: mesh_panic is the existing panic function — call `crate::panic::mesh_panic(msg_ptr)` after
      formatting the failure message. This ensures the test body unwinds and the harness catches the panic.
    - If cond != 0: do nothing (assertion passed)

    `mesh_test_assert_eq(lhs: *const MeshString, rhs: *const MeshString, expr_src: *const MeshString, _file: *const u8, _file_len: u64, _line: u32)`:
    - If lhs != rhs (string comparison): format message `"assert_eq failed: {expr_src}\n  left:  {lhs}\n  right: {rhs}"`
    - Call mesh_test_fail_msg then mesh_panic
    - Note: lhs and rhs are already converted to strings by the lowerer (Plan 03 handles type dispatch)

    `mesh_test_assert_ne(lhs, rhs, expr_src, file, file_len, line)` — symmetric: fail if lhs == rhs

    `mesh_test_assert_raises(fn_ptr: *const u8, env_ptr: *const u8, _file: *const u8, _file_len: u64, _line: u32)`:
    - Use std::panic::catch_unwind with AssertUnwindSafe to invoke the closure:
      ```rust
      // The closure ABI: fn_ptr(env_ptr) -> i64 (return value ignored)
      let result = std::panic::catch_unwind(std::panic::AssertUnwindSafe(|| {
          let f: extern "C" fn(*const u8) -> i64 = unsafe { std::mem::transmute(fn_ptr) };
          unsafe { f(env_ptr) };
      }));
      match result {
          Ok(_) => {
              // Did NOT raise — test failure
              // Format error message and call fail_msg + mesh_panic
          }
          Err(_) => {
              // Raised — assert_raises passes (do nothing, test continues normally)
          }
      }
      ```

    `mesh_test_summary(passed: i64, failed: i64, elapsed_ms: i64)`:
    - If there are any accumulated failures in FAIL_MESSAGES, reprint them all first:
      ```rust
      FAIL_MESSAGES.with(|fm| {
          let messages = fm.borrow();
          if !messages.is_empty() {
              println!("\n{BOLD}Failures:{RESET}");
              for msg in messages.iter() {
                  println!("{}", msg);
              }
          }
      });
      ```
    - Then print the count summary line:
      ```rust
      if failed > 0 {
          println!("\n{RED}{BOLD}{failed} failed{RESET}, {passed} passed in {elapsed:.2}s",
              elapsed = elapsed_ms as f64 / 1000.0);
      } else {
          println!("\n{GREEN}{BOLD}{passed} passed{RESET} in {elapsed:.2}s",
              elapsed = elapsed_ms as f64 / 1000.0);
      }
      ```
    This implements the user decision: "failures print inline as they happen, then a summary of
    all failures is reprinted at the bottom after the run."

    `mesh_test_cleanup_actors()` — placeholder for Plan 03:
    - Drain MOCK_ACTOR_PIDS thread_local
    - For each Pid: call `crate::actor::mesh_actor_exit(pid)` (check exact function name in actor/mod.rs)
    - If no actor exit function available, clear the list (Plan 03 will add proper cleanup)

    **Important:** Do NOT call `process::exit()` anywhere in test.rs. The harness (lowered in Plan 03)
    calls `std::process::exit(if fail_count > 0 { 1 } else { 0 })` as the last statement of main.
