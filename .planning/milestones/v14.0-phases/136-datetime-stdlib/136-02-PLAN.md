DateTime API surface (from Plan 01 — now compiled into mesh-rt):
- DateTime.utc_now() -> DateTime
- DateTime.from_iso8601(s: String) -> Result<DateTime, String>
- DateTime.to_iso8601(dt: DateTime) -> String
- DateTime.from_unix_ms(ms: Int) -> Result<DateTime, String>
- DateTime.to_unix_ms(dt: DateTime) -> Int
- DateTime.from_unix_secs(s: Int) -> Result<DateTime, String>
- DateTime.to_unix_secs(dt: DateTime) -> Int
- DateTime.add(dt: DateTime, n: Int, unit: String) -> DateTime
- DateTime.diff(dt1: DateTime, dt2: DateTime, unit: String) -> Float
- DateTime.before?(dt1: DateTime, dt2: DateTime) -> Bool
- DateTime.after?(dt1: DateTime, dt2: DateTime) -> Bool

Known output format: to_iso8601 always produces "YYYY-MM-DDTHH:MM:SS.mmmZ" (milliseconds + Z)
ISO 8601 "2024-01-15T10:30:00+05:30" normalizes to "2024-01-15T05:00:00.000Z"
