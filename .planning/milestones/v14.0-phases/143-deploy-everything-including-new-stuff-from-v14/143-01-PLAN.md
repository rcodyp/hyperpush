  <action>
    Create three files inside `registry/` for Fly.io deployment.

    **registry/.dockerignore** — CRITICAL: must exclude target/ to prevent 500MB+ upload:
    ```
    target/
    Cargo.lock
    .env
    .env.*
    ```

    **registry/Dockerfile** — Multi-stage cargo-chef pattern (standard for Rust on Fly.io).
    The registry is its own Cargo workspace (separate from the main mesh workspace to avoid
    sqlite3 link conflict). Build context is registry/ only. Binary name is `mesh-registry`.
    ```dockerfile
    FROM lukemathwalker/cargo-chef:latest-rust-1 AS chef
    WORKDIR /app

    FROM chef AS planner
    COPY . .
    RUN cargo chef prepare --recipe-path recipe.json

    FROM chef AS builder
    COPY --from=planner /app/recipe.json recipe.json
    RUN cargo chef cook --release --recipe-path recipe.json
    COPY . .
    RUN cargo build --release --bin mesh-registry

    FROM debian:bookworm-slim AS runtime
    WORKDIR /app
    RUN apt-get update && apt-get install -y ca-certificates && rm -rf /var/lib/apt/lists/*
    COPY --from=builder /app/target/release/mesh-registry /usr/local/bin/
    EXPOSE 3000
    ENTRYPOINT ["/usr/local/bin/mesh-registry"]
    ```

    **registry/fly.toml** — App config. Region: iad (US East, Claude's discretion per CONTEXT.md).
    `auto_stop_machines = false` because the registry must be always-on for package installs.
    `min_machines_running = 1` ensures no cold starts.
    ```toml
    app = "mesh-registry"
    primary_region = "iad"

    [build]
      dockerfile = "Dockerfile"

    [http_service]
      internal_port = 3000
      force_https = true
      auto_stop_machines = false
      min_machines_running = 1
    ```

  <what-needs-doing>
    Provision all cloud infrastructure before the registry can be deployed. This requires
    CLI tools and dashboard access that only you can authenticate with.

    **Prerequisites: Install tools if missing**
    ```bash
    # Fly CLI
    brew install flyctl   # or: curl -L https://fly.io/install.sh | sh

    # Wrangler (Cloudflare CLI)
    npm install -g wrangler
    ```

    **Step 1: Create Cloudflare R2 bucket** (do this FIRST — credentials needed for Step 3)
    ```bash
    # Authenticate with Cloudflare if needed
    npx wrangler login

    # Create the R2 bucket
    npx wrangler r2 bucket create mesh-packages

    # Verify
    npx wrangler r2 bucket list
    ```
    Then go to: Cloudflare Dashboard → R2 Object Storage → Manage R2 API tokens
    Create an API token with "Object Read and Write" permissions on the mesh-packages bucket.
    Save the Access Key ID and Secret Access Key.
    Also note your Cloudflare Account ID (visible in the dashboard URL or Overview page).

    **Step 2: Create GitHub OAuth App for production**
    Go to: GitHub → Settings → Developer settings → OAuth Apps → New OAuth App
    - Application name: Mesh Registry
    - Homepage URL: https://packages.meshlang.dev
    - Authorization callback URL: https://api.packages.meshlang.dev/auth/callback
    Save the Client ID and generate a Client Secret.

    **Step 3: Provision Fly.io app and database**
    ```bash
    # Authenticate with Fly
    flyctl auth login

    # From inside the registry/ directory:
    cd /Users/sn0w/Documents/dev/mesh/registry

    # Create the Fly app (--no-deploy so we can set secrets first)
    fly launch --name mesh-registry --region iad --no-deploy

    # Create and attach managed Postgres (same region for low latency)
    fly postgres create --name mesh-registry-db --region iad
    fly postgres attach mesh-registry-db --app mesh-registry
    # DATABASE_URL is now auto-set as a Fly secret — do NOT set it manually

    # Set all remaining secrets (replace placeholder values with real credentials from Steps 1 and 2)
    fly secrets set \
      STORAGE_ENDPOINT="https://YOUR_CLOUDFLARE_ACCOUNT_ID.r2.cloudflarestorage.com" \
      STORAGE_BUCKET="mesh-packages" \
      STORAGE_ACCESS_KEY_ID="YOUR_R2_ACCESS_KEY_ID" \
      STORAGE_SECRET_ACCESS_KEY="YOUR_R2_SECRET_ACCESS_KEY" \
      STORAGE_REGION="auto" \
      GITHUB_CLIENT_ID="YOUR_GITHUB_OAUTH_APP_CLIENT_ID" \
      GITHUB_CLIENT_SECRET="YOUR_GITHUB_OAUTH_APP_CLIENT_SECRET" \
      GITHUB_CALLBACK_URL="https://api.packages.meshlang.dev/auth/callback" \
      SESSION_SECRET="$(openssl rand -hex 32)" \
      --app mesh-registry
    ```

    **Step 4: DNS records** (add in your DNS provider)
    - `api.packages.meshlang.dev` → CNAME → `mesh-registry.fly.dev`
    - `packages.meshlang.dev` → will be configured automatically by Cloudflare Pages (Plan 04)

    **Step 5: Add custom domain to Fly (triggers TLS cert)**
    ```bash
    fly certs add api.packages.meshlang.dev --app mesh-registry
    # Fly will confirm the CNAME target — verify it matches what you added to DNS
    ```
