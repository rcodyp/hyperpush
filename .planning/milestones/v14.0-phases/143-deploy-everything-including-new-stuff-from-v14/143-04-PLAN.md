  <action>
    This task requires Cloudflare Pages dashboard Git integration — first-time project setup
    cannot be done via wrangler CLI. The user must complete this in the Cloudflare dashboard.

    The SvelteKit app was built in Plan 02 and is in packages-website/ in the repo.
    Push must be on main branch for CF Pages to pick it up.

    Steps:
    1. Go to Cloudflare Dashboard → Workers and Pages → Create application → Pages → Connect to Git
    2. Select the mesh-lang/mesh repository
    3. Set build configuration: root=packages-website, command=npm run build, output=.svelte-kit/cloudflare, NODE_VERSION env var=20
    4. Click Save and Deploy — wait for first build (~2 min)
    5. In project settings → Custom domains → add packages.meshlang.dev → Activate domain
    6. Verify https://packages.meshlang.dev loads

    See what-needs-doing below for exact step-by-step instructions.
  </action>

    <automated>curl -sf https://packages.meshlang.dev > /dev/null && echo "packages site OK" || echo "Not yet live — complete CF Pages setup first"</automated>
  </verify>
  <done>
    https://packages.meshlang.dev loads successfully with "Mesh Packages" heading.
    Cloudflare Pages project "mesh-packages" is connected to mesh-lang/mesh repo on main branch.
  </done>
  <what-needs-doing>
    The SvelteKit packages website needs to be connected to Cloudflare Pages via the dashboard
    Git integration — this cannot be done via CLI alone (CF Pages requires dashboard setup for
    first-time Git integration).

    **Step 1: Create Cloudflare Pages project**
    Go to: Cloudflare Dashboard → Workers and Pages → Create application → Pages → Connect to Git

    Select the mesh-lang/mesh repository.

    Build settings:
    - Project name: `mesh-packages`
    - Production branch: `main`
    - Root directory: `packages-website`
    - Build command: `npm run build`
    - Build output directory: `.svelte-kit/cloudflare`
    - Environment variables: NODE_VERSION = 20

    Click "Save and Deploy". Wait for the first build to complete (~2 minutes).

    **Step 2: Add custom domain**
    In the Cloudflare Pages project settings → Custom domains → Add a custom domain:
    - Enter: `packages.meshlang.dev`
    - Cloudflare will automatically add the required DNS record (since both the domain and
      Pages are in the same Cloudflare account). Click "Activate domain".

    Wait for the domain to activate (usually immediate for Cloudflare-managed domains).

    **Step 3: Verify**
    Visit https://packages.meshlang.dev — you should see the "Mesh Packages" homepage.
    It may show "No packages published yet." if the registry is empty — that is expected.

    If the page fails to load registry data:
    - Check that api.packages.meshlang.dev is responding: `curl -s https://api.packages.meshlang.dev/api/v1/packages`
    - CORS errors in browser console: the registry uses CorsLayer::permissive() which allows
      all origins — SSR routes in +page.server.js fetch server-side so CORS does not apply.
  </what-needs-doing>
  <resume-signal>
    Type "deployed" when packages.meshlang.dev is live and loading, or describe any issues encountered.
