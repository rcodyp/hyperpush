2. Add Registry variants to the Dependency enum. CRITICAL: ordering must be RegistryShorthand FIRST (before Git and Path) because serde's untagged deserialization tries variants in declaration order. A bare string "1.0.0" must match RegistryShorthand before Git or Path are tried:
```rust
#[derive(Debug, Clone, Deserialize)]
#[serde(untagged)]
pub enum Dependency {
    // Must be FIRST: matches bare string "1.0.0"
    RegistryShorthand(String),
    // Must be SECOND: matches { version = "1.0.0" } tables
    Registry {
        version: String,
    },
    // Git: requires "git" key — no ambiguity with Registry
    Git {
        git: String,
        #[serde(default)]
        rev: Option<String>,
        #[serde(default)]
        branch: Option<String>,
        #[serde(default)]
        tag: Option<String>,
    },
    // Path: requires "path" key
    Path {
        path: String,
    },
}
```

3. Add a helper method to Dependency:
```rust
impl Dependency {
    /// Returns the version string if this is a registry dependency.
    pub fn registry_version(&self) -> Option<&str> {
        match self {
            Dependency::RegistryShorthand(v) => Some(v),
            Dependency::Registry { version } => Some(version),
            _ => None,
        }
    }

    /// Returns true if this is a registry (not git or path) dependency.
    pub fn is_registry(&self) -> bool {
        self.registry_version().is_some()
    }
}
```

4. Add new tests to the existing test module:
- `parse_registry_shorthand`: `foo = "1.0.0"` parses as RegistryShorthand("1.0.0")
- `parse_registry_table_form`: `foo = { version = "1.0.0" }` parses as Registry { version: "1.0.0" }
- `parse_mixed_dependency_types`: manifest with all four dependency types (registry shorthand, registry table, git, path)
- `parse_license_field`: `[package]` with `license = "MIT"` parses correctly; without license field still parses (Option)

All existing tests must continue to pass unchanged.
